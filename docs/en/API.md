Local base URL: `http://localhost:3001`. Production clients normally use same-origin `/api/...` through Nginx.

All JSON requests should send `Content-Type: application/json`. Successful and error responses include an `x-request-id` header generated by the API unless the caller supplies one.

## Health

### `GET /health`

```json
{ "status": "ok" }
```

Nginx also exposes `GET /healthz` for the web container. It is not an Express route.

## Tournament endpoints

| Method | Path | Response |
|---|---|---|
| `GET` | `/api/tournament/teams` | `{ "teams": Team[] }` with ownership information |
| `GET` | `/api/tournament/knockout` | `{ "knockout": KnockoutRound[] }` |
| `GET` | `/api/tournament/fixtures` | `{ "fixtures": Fixture[] }` |
| `GET` | `/api/tournament/table` | `{ "standings": GroupStanding[], "companyPicks": CompanyPick[] }` |
| `GET` | `/api/tournament/news` | `{ "news": NewsItem[] }` |

Live football endpoints may return mapped API-Football data or static fallback data. News returns an empty list when its RSS provider cannot be read.

## Participant endpoints

### `GET /api/participant/session/:deviceId`

Returns the participant associated with the device, or `null`.

```json
{
  "participant": {
    "deviceId": "browser-generated-id",
    "displayName": "Alice",
    "teamCode": "THA"
  }
}
```

### `POST /api/participant/select`

Creates or replaces the selection for the supplied device. Returns HTTP `201`.

```json
{
  "deviceId": "browser-generated-id",
  "displayName": "Alice",
  "teamCode": "THA"
}
```

### `POST /api/participant/change`

Changes an existing participant's team. The request body matches `/select`. Returns `404` when the device has no participant.

### `PATCH /api/participant/display-name`

```json
{
  "deviceId": "browser-generated-id",
  "displayName": "New name"
}
```

### `POST /api/participant/reset`

Deletes the server-side selection for the device.

```json
{ "deviceId": "browser-generated-id" }
```

Successful response:

```json
{ "ok": true }
```

## Validation

- `deviceId` is required and must not exceed 128 characters.
- `displayName` is required and must not exceed 80 characters.
- `teamCode` is required, converted to uppercase, and must be exactly three letters.
- The team code must exist in the application's team data.

## Error format

```json
{
  "code": "SELECTION_CONFLICT",
  "message": "Selected team is no longer available.",
  "requestId": "..."
}
```

| Status | Typical code | Meaning |
|---:|---|---|
| `400` | `INVALID_REQUEST`, `UNKNOWN_TEAM` | Invalid payload or team |
| `404` | `PARTICIPANT_NOT_FOUND`, `NOT_FOUND` | Missing participant or route |
| `409` | `SELECTION_CONFLICT` | Team already owned by another participant |
| `503` | `UPSTREAM_UNAVAILABLE` | External fetch failed |
| `500` | `INTERNAL_ERROR` | Unexpected server error |

## Not active

`src/routes/push.ts` exists, but the active server does not mount it. `/api/push/*` and `/ws` must not be treated as supported API contracts yet.
