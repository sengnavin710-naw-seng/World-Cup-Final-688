# Fixtures Live Clock Design

## เป้าหมาย

หน้า Fixtures แสดงเวลาการแข่งขันสดแบบนาฬิกาที่เดินทุกวินาที และแสดงเวลาทดเวลาที่กรรมการประกาศ เช่น `45:01 +6` โดยไม่ต้องรอ API ทุกวินาที

## รูปแบบการแสดงผล

- ระหว่างการแข่งขัน แสดง `นาที:วินาที` เช่น `34:08`
- เมื่อ API ส่งเวลาทดเวลา (`extra`) มากกว่า 0 ให้แสดงต่อท้าย เช่น `45:01 +6`
- ค่า `+6` คงที่ตามข้อมูลล่าสุดจาก API ขณะที่ clock ด้านซ้ายเดินต่อทุกวินาที
- เมื่อได้รับ fixture ชุดใหม่จาก API ให้ตั้ง clock จาก `statusElapsed` ใหม่ เพื่อลดการคลาดเคลื่อน
- สถานะพักครึ่งหรือสถานะที่การแข่งขันไม่กำลังเดินจะไม่เพิ่มวินาทีต่อ

## การส่งข้อมูล

API จะอ่าน `fixture.status.extra` จาก API-Football แล้วส่งออกเป็น `statusExtra` พร้อม `statusElapsed`, `statusShort` และ `statusLong` โดยชนิดข้อมูลทั้ง API และเว็บรองรับ `number | null`

## Clock ฝั่งเว็บ

FixturesTab จะเก็บเวลาที่ผ่านไปในหน่วยวินาที โดยเริ่มจาก `statusElapsed * 60` เมื่อได้รับข้อมูล fixture สด จากนั้นเพิ่มหนึ่งวินาทีด้วย interval เฉพาะสถานะที่บอลกำลังแข่งขัน การได้รับข้อมูลใหม่จะ reset ค่าฐานและเริ่มนับต่อ

## การทดสอบ

- API service mapping ส่ง `status.extra` เป็น `statusExtra`
- หน้า Fixtures เริ่มแสดงเวลาพร้อมวินาที
- fake timer ทำให้เวลาขยับทีละวินาที
- เวลาทดเวลาแสดงในรูป `45:01 +6`
- สถานะที่หยุดการแข่งขันไม่เดินต่อ
